# 작업 규칙

## 코드 수정 전 논의 필수
- 코드를 수정하기 전에 반드시 어떤 방향으로 수정할지 선택지를 제시하고 사용자와 논의할 것
- 사용자가 방향을 결정한 후에만 코드 수정을 시작할 것
- "해결해", "수정해" 같은 요청이 와도 바로 코드를 건드리지 말고, 먼저 현재 상황 분석 → 선택지 제시 → 사용자 결정 → 구현 순서로 진행

## 예시 흐름
1. 문제 분석: "현재 X가 Y 때문에 안 됩니다"
2. 선택지 제시: "A 방법, B 방법, C 방법이 있습니다. 각각 장단점은..."
3. 사용자 결정 대기: "어떤 방향으로 할까요?"
4. 결정 후 구현 시작

## Bedrock AgentCore 테스트 규칙
- AgentCore Runtime 관련 코드는 로컬 테스트 금지
- BedrockAgentCoreApp이 import 시 포트 8080을 바인딩해서 로컬 환경에서 블로킹됨
- 반드시 배포 후 실제 API 호출 + CloudWatch 로그로 테스트할 것
- 로컬에서 가능한 것: 개별 함수 단위 테스트 (Bedrock 직접 호출, JSON 파싱, 스키마 검증 등)
- 배포 순서: deploy-agent.sh (Runtime) → deploy-lambda.sh (Lambda) → 프론트에서 테스트 → 로그 확인
